\section{Godiva: validation against the ICSBEP experiment}
\label{sec:godiva}

Godiva (HEU-MET-FAST-$001$) is a bare $93.71\%$-enriched
uranium metal sphere. Its fast, leakage-dominated spectrum
places no demand on the resolved-resonance region. The
ICSBEP benchmark value is the experimental critical mass,
$k_\text{eff}^\text{exp} = 1.0000 \pm 0.0010$
(1~$\sigma$)~\cite{icsbep_hmf001}. This is a \emph{physical}
benchmark with a measured uncertainty, and it is the
criterion against which the engine is validated in this
section. We report OpenMC~$0.15.3$ run on the same
ENDF/B-VII.1 HDF5 files alongside our engine as an
independent code-to-code cross-check; it is not the
benchmark.

Table~\ref{tab:keff_godiva_desktop} reports the four-provider
benchmark on the large-L3 system at rank-$5$.
Table~\ref{tab:keff_godiva_fixes} ablates the four
transport-sampling features described in
Sec.~\ref{sec:godiva_fixes}.
Table~\ref{tab:keff_godiva_ranksweep} reports a rank sweep
on the small-L3 system used for spectrum illustration;
small-L3 five-seed results are in Table~\ref{tab:keff_godiva}.

\subsection{Transport sampling features}
\label{sec:godiva_fixes}

The shipped engine implements four transport-sampling
features that, individually, account for the $246$~pcm
band between a minimal-physics baseline and the
ICSBEP-validated number reported above.
Table~\ref{tab:keff_godiva_fixes} reports an ablation in
which each feature is added on top of the previous row,
holding everything else (geometry, library, statistics)
fixed.

top of the previous one. Rank-$5$ SVD with all four

\paragraph{(n,$2$n) and (n,$3$n) routed in-generation
($-106$~pcm).} Secondary neutrons from MT$=16$ and MT$=17$
are routed through transport in the current generation
rather than appended to the next-generation source bank.
Since the engine's $k_\text{eff}$ estimator is
$\lvert\text{bank}\rvert/N_\text{source}$, banking
secondaries directly inflates the estimator by the
multiplicative-reaction rate. On Godiva that rate is
$0.00253$/source. The implementation maintains a
per-history secondary stack drained inside the transport
loop with a shared \texttt{max\_events} budget that bounds
pathological cascades.

\paragraph{Linear-linear $\mu$-CDF inversion ($-31$~pcm).}
ENDF/B-VII.1 stores all $49$ U-$235$ elastic-scatter
incident-energy distributions with \texttt{interpolation =
2} (linear-linear PDF, quadratic CDF within each bin).
The angular sampler reads the \texttt{interpolation}
attribute from HDF5 and, for linear-linear bins, inverts
the quadratic CDF by solving $ax^2 + bx + c = 0$ for the
physical root in $[0, \Delta\mu]$ (same convention as
OpenMC's \texttt{Tabular::sample}); histogram bins use the
linear inversion path.

\paragraph{MT$=91$ ENDF tabulated outgoing-energy
distribution ($-98$~pcm, largest single contribution).}
The continuum-inelastic branch samples outgoing neutron
energy from the ENDF MT$=91$ tabulated distribution at
\texttt{reaction\_091/product\_0/distribution\_0/}
\texttt{\{energy,distribution\}} (matching OpenMC, MCNP,
and Serpent). The fallback evaporation spectrum
$T = \sqrt{E^\star/a}$, $a = A/8$~MeV$^{-1}$ is used only
for nuclides where the HDF5 file lacks the tabulated
distribution.

\paragraph{MT$=16$/$17$ ENDF tabulated ($-11$~pcm).}
Multiplicative reactions receive the same tabulated
treatment as MT$=91$ for self-consistency; the impact on
Godiva is within seed noise.

\paragraph{Cross-code context.}
Running OpenMC $0.15.3$ on the same HDF5 files used by
this engine yields $k_\text{eff} = 0.99901$, i.e.\
$\Delta_\text{exp} = -99$~pcm. Both codes validate
against ICSBEP inside its experimental uncertainty from
opposite sides, with no library-level correction.

The two CPU providers agree on $k_\text{eff}$ to within
$5$~pcm ($\ll\!1\,\mathrm{SEM}_\text{combined}$). Both sit
inside the ICSBEP experimental uncertainty band of
$\pm\,100$~pcm. OpenMC $0.15.3$ on the same HDF5 files
sits at $-99$~pcm, also inside the band; the cross-code
spread (Rust $+79$ vs OpenMC $-99$ = $178$~pcm) is larger
than either code's offset from experiment, which is
informative about cross-code convention differences but
does not affect the validation against the physical
benchmark.

On the large-L3 system Godiva CPU SVD rank-$5$ at
$820$~ns/p is $1.40\times$ faster than the CPU pointwise
table at $1\,150$~ns/p. The GPU throughput rows in
Fig.~\ref{fig:throughput_godiva} use the minimal-physics
baseline because the MT$=91$ tabulated outgoing-energy
distribution is not yet ported to the CUDA kernel. On the
small-L3 system (Table~\ref{tab:keff_godiva}), SVD
rank-$2$ is $2.8\times$ faster than the pointwise table at
a $75$~pcm bias relative to its own pointwise baseline.

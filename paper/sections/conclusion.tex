\section{Conclusion}

Truncated SVD of the $\log_{10}$-transformed cross-section
matrix is a defensible drop-in \emph{representation} for
the pointwise table: for a nine-nuclide thermal PWR pin cell,
the same engine running rank-$5$ SVD vs its own pointwise
table agrees on $k_\infty$ within $6$--$47$~pcm on two
hardware tiers and agrees with OpenMC to $-1$~pcm at
ten-seed statistics after a transport-level bug audit. A
Humlicek-W4 Faddeeva evaluator, pure-Rust WMP reader, and
SVD+WMP hybrid provider all work end-to-end and reproduce
$k_\infty$ within two $\mathrm{SEM}$ of the SVD baseline; a
CUDA port of the WMP evaluator is bit-exact against the CPU
path and $7\times$ faster per lookup on a consumer GPU.

The \emph{performance} story is less one-sided.
CPU SVD rank-$5$ is $1.18\times$--$1.90\times$ faster than
the in-engine pointwise table, hardware-dependent;
GPU SVD is $1.30\times$ \emph{slower} than GPU pointwise on
the same benchmark because four clad nuclides require
summing $13$ discrete-level SVD kernels per lookup.
The hybrid is $2.06\times$ slower than CPU SVD because
WMP evaluation is $\sim\!15\times$ more expensive than the
SVD FMA per lookup.

The \emph{memory} story is a correction of our earlier
messaging. The representation-byte ratio of
$132.9\times$ (WMP + smooth-SVD vs four-channel
six-temperature pointwise) is real; the in-engine
reduction we measured is $1.06\times$, and the hybrid
engine is absolutely $4.8$--$5.1\times$ larger than the
pointwise-table baseline. The discrete-level SVD kernels,
which WMP does not cover, dominate the engine's working
set. WMP is a memory-efficient representation of the
resolved-resonance channels it covers; it is not a memory
saving at the scale of a running transport engine whose
reaction set extends far beyond those channels.

The practical takeaway for a reader considering this
approach: rank-$5$ SVD with Ducru temperature interpolation
is correct and modestly faster than pointwise on the CPU;
the $43/47$ rank-one reactions are a real
structural property of ENDF/B-VII.1 cross sections, worth
exploiting. WMP and SVD compose cleanly and produce a
correct hybrid engine, but the headline memory reduction
does not come from that combination alone in our setup;
engine-scale memory savings would require compressing the
cascade reactions too. We report the positive, mixed, and
negative findings together so the trade-off is visible.

3-phase implementation of `outputs/svd_literature_review.txt` — 2026-05-08
==========================================================================

Implementation pulls from items (A), (C), (B) of the literature review,
in that order. Each phase ships a module + a benchmark binary + unit
tests, and reports deltas against Table, SVD, and (where applicable)
WMP / per-pair-2D-SVD baselines.

All numbers are `[micro]` scope unless tagged otherwise — these are
isolated kernel measurements on synthetic and real-data matrices, not
end-to-end k_eff or transport-throughput numbers.

────────────────────────────────────────────────────────────────────────
Phase 1 — Evans 2020: SVD on adjoint flux for WW compression
────────────────────────────────────────────────────────────────────────

Implementation:
  src/random_ray/adjoint_svd.rs       — AdjointSvd type + tests (5 ✓)
  src/bin/rr_adjoint_svd.rs           — benchmark binary

Three regimes tested, results in `outputs/phase1_adjoint_svd.txt`:

[micro] Regime 1 — REAL slab adjoint, [1×1×25] reshaped [5×5]
  rank 1: 88 B (56% smaller than dense 200 B), 92% max-rel err
  rank 5: 440 B (220% LARGER), exact recon
  Reading: at 25 entries, SVD overhead dominates. No win.

[micro] Regime 2 — SYNTHETIC upscaled slab, [20×20×25] = 10000 voxels
  rank 1: 3408 B (95.7% smaller than 80 KB dense), 6.4e-16 frob err
  rank 5: 17040 B (78.7% smaller), 6.6e-16 frob err
  Reading: 23.5× compression at rank 1, machine-precision recon.
           This is the AP1000-projection regime. Evans' 10× claim is
           reproducible here at small scale — the slab's xy-symmetry
           makes ψ*(x,y,z) genuinely rank-1 in the [n_x*n_y, n_z]
           reshape, so adding voxels grows U linearly without adding
           singular values.

[micro] Regime 3 — REAL multigroup pin-cell, [100 × 2 groups]
  rank 1: 824 B (48.5% smaller than 1.6 KB dense), 5.4% frob err
  rank 2: 1648 B (3% LARGER), exact recon
  Reading: n_groups=2 is too thin for SVD to win on storage. C5G7-
           class problems with 7+ groups would shift the breakeven.

Delta against existing methods:
  - Dense WW JSON (`outputs/cadis_water_*.json`): SVD wins at scale ≥
    ~20×20 transverse mesh; loses at the existing 1×1×n_z layout.
  - Collision-density proxy (shield_slab --cadis-calibration): the
    SVD-reconstructed `counts` array is byte-identical to the random-
    ray dense baseline at full rank, so the downstream WW pipeline is
    unchanged. This is the plug-compatibility deliverable.
  - WMP: orthogonal — applies to XS storage, not WW.

Phase 2 — Tensor-train decomposition for (E × T × reaction)
────────────────────────────────────────────────────────────────────────

Implementation:
  src/tt_decompose.rs                 — Tt3 type + 4 unit tests ✓
  src/bin/tt_xs_bench.rs              — benchmark binary

Two runs, results in `outputs/phase2_tt_decompose.txt`:

[micro] Synthetic, (5000, 9, 52), true CP rank 5
  rank 1: TT 5061 / per-pair 260520 entries → 51.5× win on memory
          TT recon err 0.39 frob_rel, per-pair 0.21 — TT worse
  rank 5: TT 25485 / per-pair 1302600 → 51.1× win
          TT recon 3.5e-14, per-pair 9.3e-13 — both essentially exact

[micro] Real U-235, (3000, 6, 6) — MT={2,16,17,18,51,102}
  rank 1: TT 3012 / per-pair 18042 → 6.0× win
          TT err 0.24 frob_rel, per-pair 3.8e-3 — TT 60× worse
  rank 5: TT 15180 / per-pair 90210 → 5.9× win
          TT err 4.8e-3, per-pair 4.2e-6 — TT 1000× worse

Delta against existing methods:
  - Per-pair 2D SVD (current production): TT wins memory by ~n_R
    asymptotically, but loses accuracy by 1–3 orders of magnitude at
    the same nominal rank on real XS data. n_R=6 is too thin for the
    memory win to outweigh the accuracy loss.
  - Table: TT at rank 5 uses 0.6–14% of dense bytes.
  - WMP: orthogonal axis — WMP attacks T analytically. Side-by-side
    byte budget reported, not directly comparable on same metric.

The literature-review hypothesis "TT may confirm the negative result
with a stronger method" — confirmed. TT does not beat per-pair 2D SVD
at the accuracies needed for k_eff-relevant XS reconstruction. Memory
win exists but is not deployable at the rank where accuracy converges.

────────────────────────────────────────────────────────────────────────
Phase 3 — DCT-DB-style 2D-DCT compression vs SVD off-library
────────────────────────────────────────────────────────────────────────

Implementation:
  src/dct_compress.rs                 — Dct2 type + 3 unit tests ✓
  src/bin/dct_xs_bench.rs             — head-to-head binary

[micro] Real U-235 fission MT=18, (2000, 6) log-XS matrix, 94 KB dense
T_off = 750 K (between 600 K and 900 K library cols).

Reference: linear T-interpolation of the dense matrix at 750 K.

  budget   SVD k  SVD bytes  on-lib err   off-lib err   DCT K   DCT bytes  on-lib err   off-lib err
   8 kB      1    16056       1.72e-2      4.75e-3       500     8000       4.35e-2      4.15e-2
  16 kB      1    16056       1.72e-2      4.75e-3      1000    16000       2.57e-2      2.13e-2
  32 kB      1    16056       1.72e-2      4.75e-3      2000    32000       1.15e-2      7.10e-3
  64 kB      3    48168       9.08e-4      6.11e-4      4000    64000       4.03e-3      3.18e-3
 128 kB      6    96336       7.7e-16(ex)  6.3e-16(ex)  8000   128000       9.94e-4      1.67e-3

(SVD bytes don't fit budget exactly because each rank costs
(n_e + 1 + n_t) · 8 bytes — ranks 1, 3, 6 land at 16 KB / 48 KB / 96 KB.)

Reading: SVD wins decisively at every budget. At 8–32 KB, DCT off-lib
error is 6–10× larger than SVD's. At 64–128 KB, SVD becomes machine-
exact while DCT stays at 1e-3 relative error. The lit-review hypothesis
"DCT-DB might eliminate SVD's off-library memory edge" is rejected for
U-235 fission MT=18 in this regime.

Delta against existing methods:
  - Table (dense): exact at 94 KB. SVD-6 matches at 96 KB; DCT-8000
    is 1000× worse at 128 KB.
  - SVD (production rank 5): off-library 6.1e-4, dwarfs DCT-4000 at
    same budget.
  - WMP: not benchmarked here — WMP is analytical Doppler broadening,
    different axis. Combined comparison would need a new binary.

The off-library memory-edge claim from `paper §memprec` survives this
head-to-head with a clean 2D-DCT alternative. A more sophisticated
DCT-DB variant (heat-equation pre-broadening) might do better, but
the simple transform-and-truncate baseline is not competitive.
